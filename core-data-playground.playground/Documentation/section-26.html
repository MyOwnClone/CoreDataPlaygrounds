Until this point, the context has remained "empty", per se. When the context is saved it will send out notifications about the object state changes. As an optional step, we can create a NotificationListener and subscribe to the context notifications. In an iOS app, this object would likely correspond to a Fetched Results Controller or another object on your data model. In a Playground setup, we'll use printf() debugging to peak behind the scenes and examine the notifications sent by Core Data.`
